GeoLM Model:

The GeoLM model, developed by Knowledge Computing lab, is designed to identify toponyms (i.e., place names) and relations within sentences.

eg: I explored Rome in Georgia to visit Berry College.

GeoLM-Linking:

My responsibility involves implementing GeoLM-Linking, which includes the integration of the GeoLM model into the front end and user interaction . This implementation aims to display the probability distribution of potential locations in a visual map format.

This is easily constructed using Streamlit and deployed on Hugging Face.

The goal is to create an easily understandable application with a simple interface. The application will feature only one input box where users can enter a sentence. Once the user submits the sentence, the system will process it and display the detected location names on a bar menu for the user to view and select.





Approaches & data processing:


Step 1: Generate geo-entity embeddings from the input sentence using the GeoLM model. When a user selects a location, GeoLM-Linking will:
  • Retrieve the top 50 locations with the same name from the GeoNames database.
  • Store the geographical information of these items.

Step 2: For each location obtained in Step 1, do the following:
  • Search for its 50 nearest neighbors in the GeoNames database.
  • Generate candidate geo-entity embeddings using the GeoLM model.

since the GeoNames database only contains latitude and longitude information, I need to calculate the haversine distance for all the places in the database to the target place in order to find the 50 nearest neighbors of a target value









Reflection
