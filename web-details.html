Independently developed the Palette API website for the public to facilitate access to Palette, a text spotting model developed by the Knowledge Computing Lab. Successfully demonstrated a partial aspect of the project at ACM SIGSPATIAL 2023.
● Designed the UI with Figma and integrated it into React. Besides that, I used Node.js to build RESTful API connect with the front-end. Front-end manages user registration, login, and access control, with data stored in a MySQL database. This ensures that only authorized users can access specific features and obtain API keys.
● Incorporated the Palette model into the API and efficiently deployed the entire system on AWS using Docker for packaging.

Crafting a Quick Prototype from my own Experience

I arranged the information in a structured way, splitting the website into four parts: Documentation, API Key, Login, and SignUp. Even though I quickly made the first version in an hour, I spent another week integrating it into React.

Quick phototype

Documentation

I use the menu bar to categorize the page into two sections: the documentation page and the API key page . The documentation page is accessible without authentication. However, the API key page requires user login. If a user attempts to access the API key page without being logged in, they will be redirected to the login page. Once logged in, users can obtain an API key from the API key page and use it in the Demo section of the Documentation page.

On the API key page, I aimed for simplicity. I placed the "Create API" button at the center for easy key creation and used a table to display key information. Users can conveniently copy and paste the key for demo purposes or add it to try out the scripts provided in the documentation for sending requests to our server.

Once the primary features have been implemented, to enhance user experience and conduct thorough testing, I launched a survey reaching over 20 participants, comprising both lab members and friends. The goal was to collect data on their usual information-seeking habits, along with their feedback on the initial prototype. Additionally, I sought insights on how the prototype stacked up against other API websites like OpenAI API and Google Vision API. Analyzing the responses provided valuable guidance for the project's trajectory.

More than half of the respondents found the functionality unclear. In response, I separated the Demo section from the Documentation page and created a new page for Demo. This makes it easier for users to access the Demo and aligns better with the demo requirements for the upcoming conference.

Survery Question 2

Respondents with neutral and negative opinions expressed the need for clearer navigation, specifically suggesting the addition of login and signup buttons on the menu bar. Taking this feedback into account, I redesign the menu bar.

Iteration


Iteration & Deliverable

In user testing, participants gave valuable feedback on the redesigned features. They liked the improved navigation through the menu bar but found the original login and SignUp confusing. Taking this valuable suggestion into account, I redesigned the menu bar to enhance both its aesthetics and user-friendliness. Now, there are three distinct menu bars for non-logged-in users, User Logged In, and administrator Logged In .

Reflection

The ACM SIGSPATIAL 2023 accepted the paper in September, with the demo scheduled for Nov 15th. The tight timeframe posed a challenge to independently create a full-stack website, spanning from design to development. Unlike a traditional research and design process, I didn't have enough time to conduct extensive research beforehand. Despite this, I still finished it with a commitment to enhancing my design and skills through iterative processes.

In this project, user experience is not limited to the front-end design but extends to the back-end development as well. For example: To enhance security, I utilized JSON Web Tokens to send a refresh token to the frontend instead of storing sensitive user information, such as passwords, in cookies. Additionally, Since the model's output is in the form of a GeoJson file. To make this result visible to users in front-end, I wrote code to draw text onto the original image from Json format result.

Another aspect of the project involved deploying the entire system efficiently on Amazon Web Services using Docker for packaging. This experience deepened my understanding of containerization and cloud technologies. I truly appreciate it. While proud of this project, I believe I'll do even better in my next one.
